Terminology

The Terminology module uniquely and clearly defines the key terms used in public administration. It provides solutions to semantic barriers, which may includeIn

  • different definitions of concepts,
  • the same term for several concepts or several terms for the same concept,
  • describing the same concept with different attributes or with a different number of attributes,
  • the use of different units,
  • different accuracy of measurement scales.
  • etc.

In the Terminology module, all terms have an unambiguous and non-redundant definition. The minimum it provides is:

  • if a term is used for different concepts and in different contexts, it must explicitly be qualified in a certain context,
  • if several terms are used for the same concept, one must be designated as preferred, and the others as synonyms, pseudonyms, etc.

Terms can also be organised in a hierarchical structure. Each term can have one or more parent or child relationships to other terms. A network structure is also allowed (a term can have several parent terms). Relationships between terms can also include associative (non-hierarchical) relationships.

The Terminology module also contains other metadata

The Terminology module is interelated and coordinated with the Data Models and Reference Data (Code lists) module.

Terminologija

V modulu Terminologija so enolično in jasno definirani ključni pojmi, ki se uporabljajo v okviru javne uprave. Modul omogoča rešitve za semantične ovire, ki so lahko:

  • različne definicije pojmov,
  • isti termin za več pojmov ali več terminov za isti pojem,
  • opis istega pojma z različnimi atributi ali z različnim številom atributov,
  • uporaba različnih enot,
  • različna natančnost merskih lestvic,
  • itd.

V modulu Terminologija imajo vsi pojmi nedvoumno in neredundantno definicijo. Najmanj, kar zagotavlja, je:

  • če se nek termin uporablja za različne pojme in v različnih kontekstih, je v določenem kontekstu eksplicitno kvalificiran,
  • če se uporablja več terminov za isti pojem, je eden določen kot prednostni, ostali pa kot sinonimi, psevdonimi itd.

Termini so lahko organizirani tudi v hierarhično strukturo. Vsak termin je lahko v enem ali več odnosih nadrejenosti ali podrejenosti do drugih terminov. Dovoljena je tudi mrežna struktura (en termin ima lahko več nadrejenih terminov). Odnosi med pojmi lahko vključujejo tudi asociativne (nehierarhične) odnose.

Modul Terminologija vsebuje tudi druge metapodatke (npr. izvorno vsebinsko področje, skrbništvo pojmov itd.)

Modul Terminologija se vzajemno povezuje in usklajuje z moduloma Podatkovni modeli in Šifranti.
